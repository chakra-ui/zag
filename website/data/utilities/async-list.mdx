The async list is used to display a list of items that are loaded
asynchronously. Usually paired with the combobox or select component.

> This was inspired by React Stately's
> [useAsyncList](https://react-spectrum.adobe.com/react-stately/useAsyncList.html)
> hook.

<Resources pkg="@zag-js/async-list" />

**Features**

- Support for pagination, sorting, and filtering
- Support for abortable requests via `AbortSignal`
- Manages loading and error states

## Usage

First, import the async list package into your project

```jsx
import * as asyncList from "@zag-js/async-list"
```

The async list package exports two key functions:

- `machine` — The state machine logic for the async list widget.
- `connect` — returns the properties and methods for the async data management.

<CodeSnippet id="async-list/usage.mdx" />

### Loading and Error States

The async list machine will automatically return the `error` and `loading`
properties in the `api` when the `load` function returns an error or is loading.

- `api.error` — The error instance returned by the last fetch.
- `api.isLoading` — Whether the list is loading (initial load, reload, filter,
  or sort).
- `api.isLoadingMore` — Whether more items are being loaded via pagination.
- `api.isEmpty` — Whether the list has no items.
- `api.items` — The items in the list after the last fetch.

### Pagination

The async list supports paginated data to avoid loading too many items at once.
This is accomplished by returning a cursor in addition to items from the `load`
function.

When `loadMore` is called, the cursor is passed back to your `load` function,
which you can use to determine the URL for the next page.

```tsx
const service = useMachine(asyncList.machine, {
  async load({ signal, cursor }) {
    const requestUrl = cursor || "https://pokeapi.co/api/v2/pokemon"
    const res = await fetch(requestUrl, { signal })
    const json = await res.json()

    return {
      items: json.results,
      cursor: json.next,
    }
  },
})
```

Then, you can use the `api.loadMore` method to load more items.

```tsx
api.loadMore()
```

### Reloading the data

Use the `api.reload` method to reload the data.

```tsx
api.reload()
```

### Sorting

The async list machine supports both client-side and server-side sorting. You
can implement sorting by providing a `sort` function for client-side operations,
or by handling the `sorting` parameter in your `load` function for server-side
sorting.

Regardless of the sorting implementation, the way to trigger the sorting is by
calling the `api.setSorting` method with the descriptor.

```tsx
api.setSorting({ column: "name", direction: "ascending" })
```

#### Client-side sorting

Use the `sort` function to implement client-side sorting.

```tsx
const service = useMachine(asyncList.machine, {
  async load({ signal }) {
    // ...
  },
  sort({ items, sorting }) {
    return {
      items: items.sort((a, b) => {
        // Compare the items by the sorted column
        const aColumn = a[sorting.column]
        const bColumn = b[sorting.column]
        let direction = aColumn.localeCompare(bColumn)

        // Flip the direction if descending order is specified.
        if (sorting.direction === "descending") {
          direction *= -1
        }

        return direction
      }),
    }
  },
})
```

#### Server-side sorting

For server-side sorting, use the `sorting` parameter in the `load` function to
pass sorting parameters to your API.

```tsx
const service = useMachine(asyncList.machine, {
  async load({ signal, sorting }) {
    let url = new URL("http://example.com/api")
    if (sorting) {
      url.searchParams.append("sort_key", sorting.column)
      url.searchParams.append("sort_direction", sorting.direction)
    }

    let res = await fetch(url, { signal })
    let json = await res.json()
    return {
      items: json.results,
    }
  },
})
```

### Filtering

Filtering your data list is often necessary, such as for user searches or query
lookups. For server-side filtering, use the `filter` parameter in the `load`
function.

The way to trigger the filtering is by calling the `api.setFilter` method with
the filter value.

```tsx
api.setFilter("filter text")
```

The `api.setFilter` method modifies the `filter` and calls the `load` function
to refresh the data with the updated filter.

```tsx
const service = useMachine(asyncList.machine, {
  async load({ signal, filter }) {
    let url = new URL("http://example.com/api")
    if (filter) {
      url.searchParams.append("filter", filter)
    }

    let res = await fetch(url, { signal })
    let json = await res.json()
    return {
      items: json.results,
    }
  },
})
```

### Aborting requests

Use the `api.abort` method to abort the current request.

```tsx
api.abort()
```

This only works if you pass the `signal` parameter to the `load` function.

```tsx
const service = useMachine(asyncList.machine, {
  async load({ signal }) {
    // ...
  },
})
```

### Registering external dependencies

In even more complex scenarios, you may need to register external dependencies
that trigger a reload of the data.

Use the `dependencies` parameter to register external dependencies. Ensure every
dependency is a primitive value (no objects, arrays, maps, sets, etc.).

```tsx
const service = useMachine(asyncList.machine, {
  dependencies: [userId],
  async load({ signal }) {
    // read the external dependency directly from your scope
    const res = await fetch(`/api/users/${userId}/items`, { signal })
    return { items: await res.json() }
  },
})
```

## Composing with a collection

Async list is a data loader. It handles fetching, pagination, sorting, and
filtering, but it doesn't own the list structure. To render items in a
`combobox`, `select`, or `listbox`, pair it with a collection. Async list owns
the server data, and the collection owns the UI structure and any client-side
manipulation.

```tsx
import * as asyncList from "@zag-js/async-list"
import * as combobox from "@zag-js/combobox"
import { normalizeProps, useMachine } from "@zag-js/react"
import { useMemo } from "react"

function ComboboxWithAsyncData() {
  // 1. Load server data with async list (filtering happens on the server)
  const list = useMachine(asyncList.machine, {
    async load({ signal, filter }) {
      const res = await fetch(`/api/items?q=${filter ?? ""}`, { signal })
      return { items: await res.json() }
    },
  })
  const listApi = asyncList.connect(list)

  // 2. Build a collection from the loaded items for the combobox to render
  const collection = useMemo(
    () => combobox.collection({ items: listApi.items }),
    [listApi.items],
  )

  // 3. Drive server-side filtering from the combobox input
  const cb = useMachine(combobox.machine, {
    collection,
    onInputValueChange({ inputValue }) {
      listApi.setFilter(inputValue)
    },
  })

  // ...render the combobox
}
```

### Mutating items

Collection mutation methods (`insert`, `append`, `prepend`, `remove`, `move`,
`update`, `reorder`) are immutable and return a new collection. Keep the
collection in state, seed it from the loaded items, and apply optimistic updates
without refetching.

```tsx
const [collection, setCollection] = useState(() =>
  combobox.collection({ items: [] }),
)

// keep the collection in sync with freshly loaded data
useEffect(() => {
  setCollection(combobox.collection({ items: listApi.items }))
}, [listApi.items])

// optimistic mutations
const removeItem = (value: string) => setCollection((c) => c.remove(value))
const appendItem = (item: Item) => setCollection((c) => c.append(item))
```
